Version 6 of _iframe-resizer_ is a complete rewrite of the project in TypeScript. The entire library now ships with first‑class type definitions, the framework ecosystem has been greatly expanded, and a number of rarely used options have been removed in favor of the library's auto‑detection logic.

<Aside type="note" title="Coming from Version 4?">
  It is recommended to first follow the [Upgrade to V5 guide](https://v5.iframe-resizer.com/upgrade) before applying the changes on this page. The v5 documentation also remains available at [v5.iframe-resizer.com](https://v5.iframe-resizer.com) for reference.
</Aside>

## TypeScript First

The entire codebase has been ported to TypeScript and all packages now publish native type definitions. Types are exported directly from each package, rather than via the old `declare module 'iframe-resizer'` namespace that was shipped via DefinitelyTyped.

```ts
import iframeResize, {
  type IFrameObject,
  type IFrameOptions,
  type IFrameResizedData,
} from '@iframe-resizer/parent'
```

The [Types](/api/types) page in the API section lists all the types and constants that are exported from `@iframe-resizer/core`, `@iframe-resizer/parent`, and `@iframe-resizer/child`.

## New Framework Packages

_Iframe-resizer 6_ introduces five new framework integrations in addition to the existing React, Vue, Angular, and jQuery packages.

- [Alpine.js](/frameworks/alpine) — `@iframe-resizer/alpine`
- [Astro](/frameworks/astro) — `@iframe-resizer/astro`
- [Solid](/frameworks/solid) — `@iframe-resizer/solid`
- [Svelte](/frameworks/svelte) — `@iframe-resizer/svelte`
- [Web Component](/frameworks/web-component) — `@iframe-resizer/web-component`

The [Angular](/frameworks/angular) package is now distributed as a proper NPM package, rather than as a directive to be copied into your project.

## Breaking Changes

### React: `forwardRef` prop replaced with standard `ref`

The `@iframe-resizer/react` component now uses React's standard `ref` prop, rather than the custom `forwardRef` prop used in v5.

```diff
- <IframeResizer forwardRef={iframeRef} />
+ <IframeResizer ref={iframeRef} />
```

The shape of the ref object itself is unchanged, however the `getElement()` method is now deprecated in favor of `getRef()`, which returns a standard React ref object.

### Vue 2 support dropped

The `@iframe-resizer/vue` package now requires Vue `^3.3.0` and uses the Composition API. If you are still using Vue 2, you can follow the [Vue 2 Directive](/frameworks/vue2) guide, which uses the [Framework API](/api/framework) directly.

### Custom content size calculation methods removed

In v5, the `heightCalculationMethod` and `widthCalculationMethod` options were deprecated in favor of the new auto‑detection logic. In v6, these options and all of the custom calculation methods have been fully removed. The library now always uses `auto` mode.

The child page methods `setHeightCalculationMethod()` and `setWidthCalculationMethod()` have also been removed.

If you have custom sizing requirements, use the [`onBeforeResize()`](/api/child#onbeforeresize) event on the child page to modify or replace the calculated size before it is sent to the parent page.

### `@iframe-resizer/legacy` package removed

The legacy backwards‑compatibility bundle that was used to support v4 child pages has been removed. The v6 parent page still works with v4 child pages (via the built‑in legacy support), but the separate `@iframe-resizer/legacy` package is no longer published.

### Naming changes on `window` and iframe

The `window.iFrameResizer` and `window.parentIFrame` objects, along with the `iframe.iFrameResizer` property, have been renamed to use lowercase `f`. The old names are no longer supported.

```diff
- window.iFrameResizer = { license: 'xxxx' }
+ window.iframeResizer = { license: 'xxxx' }

- window.parentIFrame.sendMessage('hello')
+ window.parentIframe.sendMessage('hello')

- iframe.iFrameResizer.sendMessage('hello')
+ iframe.iframeResizer.sendMessage('hello')
```

### Parent-side `IFrameObject.resize()` removed

The `resize()` method was removed from the parent‑side `IFrameObject` that is attached to the iframe. Resizing is always driven by the child page, and the child page can trigger a resize manually via [`parentIframe.resize()`](/api/child#resize).

### `IFrameMouseData` shape changed

The `onMouseEnter` and `onMouseLeave` callbacks now receive an object containing `screenX` and `screenY` properties, rather than `height` and `width`.

```diff
onMouseEnter: ({ iframe, type,
-  height, width
+  screenX, screenY
}) => { }
```

### `IFrameScrollData` shape changed

The `onScroll` callback now receives `top` and `left` properties. The old `x` and `y` properties are still provided for backwards compatibility, but are deprecated and will be removed in a future release.

```diff
onScroll: ({ iframe,
-  x, y
+  top, left
}) => { }
```

### `onBeforeResize()` return type enforced

The [`onBeforeResize()`](/api/child#onbeforeresize) callback on the child page must now return a `number`. In v5 a return value of `undefined` was silently tolerated, in v6 this will be flagged by the TypeScript compiler.

### Child `resize()` parameter types

The [`parentIframe.resize()`](/api/child#resize) method now accepts `number` parameters for `customHeight` and `customWidth`. Previously these were loosely typed as strings.

## New Features

### `bodyBackground`, `bodyMargin`, and `bodyPadding` options

Three new parent page options have been added to allow you to override the default CSS of the body element in the iframe. See the [Parent Page API](/api/parent#options) for details.

### Numeric shorthand for `log`

The `log` option now accepts numeric values, which can be useful when enabling logging from a URL parameter or environment variable.

```js
iframeResize({
  log: 2, // 0 = disabled, 1 = collapsed, 2 = expanded
})
```

The constants `LOG_DISABLED`, `LOG_COLLAPSED`, and `LOG_EXPANDED` are also exported from `@iframe-resizer/core`.

### `getVersion()` method

A new `getVersion()` method is available on both the parent and child API. It returns an object containing the version of both pages, which is useful for diagnostics.

```js
iframe.iframeResizer.getVersion()
// { parent: '6.0.0', child: '6.0.0' }
```

### Multiple `targetOrigin` values

The child page [`targetOrigin`](/api/child#targetorigin) setting now accepts an array of origins, to allow the iframe to be embedded on multiple parent domains whilst still restricting `postMessage` to a known allow-list.

### Automatic `<title>` detection

Changes to the `<title>` element inside the iframe are now automatically detected and propagated to the parent page, where they are mirrored onto the iframe's `title` attribute.

## Other Improvements

- Integration tests have been expanded to cover every framework package.
- Unit test coverage has been raised to 99%.
- The build system has been migrated from Rollup to a hybrid Vite / Rollup setup.
- All packages now have consistent type definitions sourced from a single location.
