The Parent API is for use on the page containing the `<iframe>` tag. It is split into three sections. 
[Options](#options) contains settings that can be passed when calling iframe-resizer. [Events](#events) 
are triggered by updates to the iframe, or messages from the iframe. [Methods](#methods) allow you to 
interact with _iframe-resizer_ after the initial resizing.

## Options

The following options can be passed to _iframe-resizer_ on the parent page.

### bodyBackground

```
default: null
type:    string
```

Override the body background style in the iframe.

### bodyMargin

```
default: null
type:    string | number
```

Override the default body margin style in the iframe. A string can be any valid value for the CSS margin 
attribute, for example `'8px 3em'`. A number value is converted into px.

### bodyPadding

```
default: null
type:    string | number
```

Override the default body padding style in the iframe. A string can be any valid value for the CSS padding 
attribute, for example `'8px 3em'`. A number value is converted into px.

### checkOrigin

```
default: true
type:    boolean | string[]
```

When set to true, only allow incoming messages from the domain listed in the `src` property of the iframe
tag. If your iframe navigates between different domains, ports or protocols; then you will need to provide 
an array of URLs or disable this option.

### direction

```
default: 'vertical'
values: 'vertical' | 'horizontal' | 'both' | 'none'
```

Set the direction in which you want the iframe to automatically scale to the content size.

When setting this option, you may need to tell _iframe-resizer_ which page element to use to calculate the 
content size by adding one or more [`data-iframe-size`](/setup/advanced#data-iframe-size) attributes.

<Aside type="tip">
  In most cases you should set one of the dimensions of the iframe to a percentage size and then configure
  _iframe-resizer_ to control the other dimension. Selecting the `both` option will require you to manually 
  control one of the dimensions of your iframe's content size, in order to avoid unexpected results.
</Aside>

### inPageLinks

```
default: false
type:    boolean
```

When enabled, in-page linking inside the iframe and from the iframe to the parent page will be enabled.

_<sup>*</sup> This option is available in the  Professional and Business editions._

### license

```
type:    string
```

**Commercial** Enter the key provided with the [purchased](/pricing) commercial license.

**Open Source** If you are using _iframe-resizer_ on an open-source site or project, you can use it 
for free under the terms of the [GNU General Public License v3](/gpl). To accept these terms, set the license to `GPLv3`.

<Aside type="tip">
  This field can alternatively be set on the [Child Page](/api/child#license), which is useful when third parties
  host your iframe on their own sites.
</Aside>

### log

```
default: false
type:    boolean | 'collapsed' | 'expanded' | number
```

Enable logging to the browser [console](https://developers.google.com/chrome-developer-tools/docs/console#opening_the_console). The output can be set to either a `'collapsed'` or `'expanded'` view, or to a numeric shorthand:

- **`0`** or **`false`** — logging disabled
- **`1`** or **`'collapsed'`** — collapsed log group per event loop frame
- **`2`** or **`'expanded'`** — expanded log group per event loop frame

The constants `LOG_DISABLED`, `LOG_COLLAPSED`, and `LOG_EXPANDED` are also exported from `@iframe-resizer/core`. See the [Types](/api/types#constants) page.

<Aside type="tip">
  To improve the readability of the log output, _iframe-resizer_ uses [auto-console-log](https://www.npmjs.com/package/auto-console-group) to automatically group messages by each frame of the event loop.
</Aside>

### offsetSize

```
default: 0
type:    integer
```

Modify the computed size of the iframe. This is useful if the page in the iframe returns a size value 
that is consistently slightly different to how you want the iframe to be sized.

_<sup>\*</sup> This value can also be set on a per page basis in the iframe._

### scrolling

```
default: false
type:    boolean | 'omit'
```

Enable scrollbars in the iframe.

- **true** applies `scrolling="yes"`
- **false** applies `scrolling="no"`
- **'omit'** applies no `scrolling` attribute to the iframe

### tolerance

```
default: 0
type:    integer
```

Set the number of pixels the iframe content size has to change by, before triggering a resize of the iframe.

<Aside type="caution" title="">
  You should not normally need to use this option, other than for debugging. If the iframe is in a resizing
  loop, then this indicates there is an issue in your CSS, which is causing the content to always
  scale to a size larger than the current viewport. See the [Content Guidelines](/content_guidelines) section for more details on how to resolve such issues.
</Aside>

### waitForLoad

```
default: false
type:    boolean
```

By default _iframe-resizer_ sends a message to the iframe as soon as it is called, and then tries again when the
`onload` event of the iframe is triggered. Setting this option to __true__ will disable the first attempt to contact
the iframe and force it to wait for the load event to fire. This is useful with some web frameworks that create the 
iframe after the main webpage has loaded.

### warningTimeout

```
default: 5000
type:    integer
```

Set the number of milliseconds after which a warning is logged if the iframe has not responded. Set to **0** to 
suppress warning messages of this type.

## Events

The following callback events can be passed to _iframe-resizer_ on the parent page, as part of the 
[options](#options) object.

### onAfterClose

```js
onAfterClose: (iframeID) => void
```

Called after the iframe is closed via `parentIframe.close()` method.

### onBeforeClose

```js
onBeforeClose: (iframeID) => boolean
```

Called before the iframe is closed via `parentIframe.close()` method. Returning `false` will prevent the iframe 
from closing.

### onMessage

```js
onMessage: ({ iframe, message }) => void
```

Receives messages posted from the iframe with the `parentIframe.sendMessage()` method.
Passes an [`IFrameMessageData`](/api/types#iframemessagedata) object containing the **iframe** and
the **message**.

### onMouseEnter

```js
onMouseEnter: ({ iframe, screenX, screenY, type }) => void
```

Function called after the mouse enters the iframe. Passes an [`IFrameMouseData`](/api/types#iframemousedata) object containing the **iframe**,
**screenX**, **screenY** and the **type** of event that triggered the callback.

### onMouseLeave

```js
onMouseLeave: ({ iframe, screenX, screenY, type }) => void
```

Function called after the mouse leaves the iframe. Passes an [`IFrameMouseData`](/api/types#iframemousedata) object containing the **iframe**,
**screenX**, **screenY** and the **type** of event that triggered the callback.

### onReady

```js
onReady: (iframe) => void
```

Called after initial setup. Passes the [`IFrameComponent`](/api/types#iframecomponent) — the iframe element with the
`iframeResizer` control object attached.

### onResized

```js
onResized: ({ iframe, height, width, type }) => void
```

Function called after iframe resized. Passes an [`IFrameResizedData`](/api/types#iframeresizeddata) object containing the **iframe**, **height**,
**width** and the **type** of event that triggered the iframe to resize.

### onScroll

```js
onScroll: ({ iframe, top, left, }) => boolean
```

Called before the page is repositioned after a request from the iframe, due to either an in‑page link or a direct
request from either [parentIframe.scrollTo()](/api/child#scrolltox-y) or 
[parentIframe.scrollToOffset()](/api/child#scrolltooffsetx-y). Passes an
[`IFrameScrollData`](/api/types#iframescrolldata) object containing the **iframe**, **top**, and **left**.
If this function returns **false**, it will stop
the library from repositioning the page, so that you can implement your own custom scrolling instead.

The following example shows how to adjust the scroll point. This can be useful if you have a sticky header on your 
page and you need to ensure the target scroll point does not get covered by your header.

```js
const SCROLL_OFFSET = 50

iframeResize({
  onScroll: ({ top, left }) {
    window.scrollTo({
      top: top + SCROLL_OFFSET,
      left,
      behavior: 'auto'
    })

    return false // Stop iframe-resizer scrolling the page
  }
})
```

## Methods

Once the iframe has been initialized, an [`IFrameObject`](/api/types#iframeobject) is bound to it as
`iframe.iframeResizer`. The following methods are available.

### close()

Remove the iframe from the parent page. The [`onBeforeClose`](#onbeforeclose) callback is called first
and can prevent closing by returning `false`; the [`onAfterClose`](#onafterclose) callback is called once
the iframe has been removed.

### disconnect()

Disconnect _iframe-resizer_ from the iframe. This method MUST be called before removing an iframe from the page.

### getVersion()

Returns the version of both the parent and child page _iframe-resizer_ packages, as an
[`IFrameVersion`](/api/types#iframeversion) object.

```js
iframe.iframeResizer.getVersion()
// { parent: '6.0.0', child: '6.0.0' }
```

If the child page is running an older version of _iframe-resizer_ that does not report its version, the
`child` property will be `'legacy'`.

### moveToAnchor(<span class="args">anchor</span>)

Move to an anchor in the iframe.

### sendMessage(<span class="args">message, [targetOrigin]</span>)

Send data to the iframe; `message` can be any data type that can be serialized into JSON. The `targetOrigin` 
option is used to restrict where the message is sent, in case your iframe navigates to another domain.
